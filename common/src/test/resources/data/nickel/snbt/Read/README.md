# 测试：镍-SNBT 解析-合法输入

## 测试描述

本数据集验证 SNBT 读者（`SNBTReader.readNBTFromInput` / `readSingleNBTFromInput`）对合法输入的解析结果。读者从输入流读出一个复合标签，测试把结果与期望 NBT 结构做 `equals` 比较。

读者支持的语法要素：

- 复合标签 `{key:value,...}` 与列表 `[v1,v2,...]`，两者都允许尾逗号，元素之间也允许空白；
- 键有两种写法：无引号键（字符集 `[0-9A-Za-z_\-.+]`）与引号字符串键（原样读出，不剥离空白）；
- 字符串值支持单双引号，转义支持全套：`\n`、`\t`、`\\`、`\uXXXX`、`\xXX`、八进制、`\N{Unicode 名称}` 等；
- 数字字面量支持类型后缀（`b`/`s`/`l`/`f`/`d`，大小写均可）、无符号标记 `u`、十六进制前缀 `0x`、二进制前缀 `0b`、数字分隔符 `_`；
- `true` / `false` 解析为字节 1 / 0。

### 锁定的预期行为

下面这些用例锁定当前实现的宽容或意外行为。它们不是理所当然的正确，语义调整时应同步审视；用例名或注释里会标出。

- 尾逗号被接受：`{a:1,}` 与 `[1,2,]` 都能正常读出。
- 列表在探测数组前缀之前会先跳过空白，因此 `{a:[ ]}` 读成空列表。这与复合标签不对称——`{ }` 会报错（见 `ReadError/`）。
- 引号键两端的空白保留：`{"  a  ":1}` 的键就是 `  a  `。纯空白键合法；空串键报 EMPTY_KEY。
- `readNBTFromInput` 不检查复合标签之后的剩余内容，因此 `{a:1}xyz` 正常读出 `{a:1}`。`readSingleNBTFromInput` 则会检查（见 `ReadError/` 的 NO_SPLIT）。
- 首字母大写的 `True` 不是布尔值，读出字符串 `"True"`；只有全小写的 `true` / `false` 才是布尔。
- 十进制 int 溢出不报错：`2147483648` 超出 int 范围后落入浮点解析，读成 double。
- 无符号字面量按位装入对应的有符号类型：`255ub` 读出 byte -1。
- 十六进制里的 `b` 是数字位，不是 Byte 后缀，因此 `0xffb` 是 int 4091。
- `0b` 本身是 Byte 0：空内容的二进制前缀不成立，退回按十进制 `0` 加后缀 `b` 解析；`-0b` 同理是 Byte 0。
- 十六进制里的 `u` 结束数字位，其后还能再跟类型后缀：`0xffub` 是无符号 hex 装入 byte，得 -1。
- 以 `_` 开头的 token 不是数字，读成无引号字符串：`_1` 是字符串。
- 空白判定用 `Character.isWhitespace`，制表符等同空格，可以出现在键、冒号、值、逗号、括号之间的任意分隔位置。
- 空白会结束无引号 token，因此后缀相关的空白位置有讲究：
  - 数字与类型后缀之间不允许空白——`{a:1 b}` 报错（见 `ReadError/`）；
  - 后缀之后、闭括号或逗号之前的空白只是分隔——`{a:1b }` 合法。
- 进制前缀之后可以跟类型后缀，后缀字符不被当作数字位：
  - 二进制的 `0b101b` 是 Byte 5（末尾 `b` 是后缀，不是二进制位，与 hex 里 `u` 结束数字位同理）；
  - 十六进制的 `0x7fffs` 是 Short 32767；
  - 二进制的 `0b101l` 是 Long 5。
- 引号字符串里的字面制表符原样收录，不会结束引号串，与转义 `\t` 产出同一个值；引号键里的字面制表符同样原样保留。

## 样例格式

YAML 用例列表，每个用例的键：

- `name`：用例名；
- `input`：SNBT 输入原文（YAML 单引号书写）；
- `mode`：可选，取 `single` 时经 `readSingleNBTFromInput` 入口；
- `expected`：期望 NBT 结构，用"类型标注节点"描述：

```
节点 := {type: <类型名>, value: <值>}
类型名 ∈ byte | short | int | long | float | double | string | list | compound
compound 的 value 是 键 → 节点 的映射；list 的 value 是节点列表；其余为标量。
```

浮点期望值只用二进制可精确表示的数（如 1.5、100.0），或与输入十进制位完全一致的 double，以保证按位相等比较成立。

## 样例 #1

### 输入 #1

```yaml
- name: 单键整数
  input: '{a:1}'
  expected:
    type: compound
    value:
      a: {type: int, value: 1}
```

## 说明/提示

### 样例 1 解释

输入 `{a:1}` 读出含单键 `a` 的复合标签，值是无后缀十进制整数，默认类型 int。测试把 `expected` 节点转成真实 NBT 对象后，与解析结果做 `equals` 比较。

### 关于带类型前缀数组

`[B;...]` / `[I;...]` / `[L;...]` 前缀数组的合法用例不在本数据集里，由 `清汩萌.镍.snbt.TestSNBTReader` 的用例直接覆盖。`[Beef,Bee]`（B 开头但不是数组前缀，是普通字符串列表）收录在本数据集作对照。
