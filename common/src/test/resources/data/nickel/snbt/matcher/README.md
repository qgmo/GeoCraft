# 测试：镍-SNBT-NBT 匹配器端到端匹配

## 测试描述

本数据集验证 NBT 匹配器体系（`moe.qingu.nickel.nbt.matcher.NBTMatcher` 及其子类）与 SNBT 读者（`SNBTReader`）联动的端到端匹配语义。

每个用例给出两段 SNBT 字符串：`matcher` 与 `target`。测试流程是：

1. 用 `SNBTReader.readNBTFromInput` 分别把两段 SNBT 解析成 `NBTTagCompound`；
2. 用 `NBTMatcher.toMatcher` 把匹配器侧的复合标签递归转换成 `NBTCompoundMatcher`；
3. 断言 `matcher.match(target)` 的结果等于 `expected`。

### 锁定的匹配语义

以下是本数据集用例锁定的匹配语义：

- **类型严格**：数值相同但 NBT 类型不同（如 `1b` 与 `1s`）一律不匹配。
- **复合标签是子集语义**：期望的每个键都必须存在且值匹配，目标多出的键不影响；空匹配器匹配任意复合标签。
- **列表是存在量词语义**：每个期望元素都要命中目标列表中至少一个元素，顺序无关。期望经 `HashSet` 去重，`[1,1]` 等价于 `[1]`。空列表匹配器只匹配空列表——这与复合标签的"空即通配"不对称。
- **浮点是精确比较**：用 `==` 语义，无容差；由此 `-0.0` 与 `0.0` 相等。
- **嵌套深度不容差**：类型严格性沿嵌套传递。期望比目标多一层（复合/列表匹配器对上标量或浅层元素），或目标比期望多一层，都会在该层因类型不符而失败。"目标多出内容不影响"只针对同一层的多余键或多余列表元素。

## 样例格式

本目录下每个 `.yaml` 文件是一组用例。文件字段：

- `ver`：正整数，格式版本，当前为 `1`；
- `group`：字符串，用例组名，作为用例显示名的前缀；
- `cases`：用例列表，每个用例含：
  - `name`：字符串，用例名（中文，格式"主题-要点"）；
  - `matcher`：字符串，一段完整的 SNBT 复合标签，作为匹配器模板；
  - `target`：字符串，一段完整的 SNBT 复合标签，作为被匹配目标；
  - `expected`：布尔值，期望 `matcher.match(target)` 的结果。

### 正负样例配对

每个匹配成功的样例都配有至少一个应当失败的镜像样例——同结构下改值、改类型或改深度的对应失败用例。共享同一匹配器模板的成功用例可以共享同一个镜像。

### 豁免

以下三条成功用例没有失败镜像，原因是构造不出来——空匹配器 `{}` 匹配任何复合标签，而本数据集的匹配器与目标都从 SNBT 文本解析、根部必然是复合标签，因此根部的 `{}` 对任何目标都必然匹配成功，不存在能让它失败的输入：

- `compound.yaml`："空匹配器-对空目标"、"空匹配器-对非空目标通配"
- `mixed.yaml`："综合-空匹配器通配复杂目标"

空匹配器唯一会失败的场合是嵌套位置对上标量值（如 `{k:{}}` 对 `{k:1}`），该失败情形由 `compound.yaml` 的"深度-空匹配器对标量不通配"覆盖。

## 样例 #1

### 输入 #1

```yaml
ver: 1
group: 复合标签
cases:
  - name: 子集-目标多余键不影响
    matcher: '{a:1}'
    target: '{a:1,b:2,c:"extra"}'
    expected: true
  - name: 缺键-目标缺少期望键
    matcher: '{a:1,b:2}'
    target: '{a:1}'
    expected: false
```

## 说明/提示

### 样例 1 解释

第一条用例的匹配器只期望键 `a` 的值为整型 `1`；目标虽然多出 `b`、`c` 两键，但按子集语义不影响，期望结果为 `true`。第二条用例的匹配器还期望键 `b`，而目标没有该键，期望结果为 `false`。

### 文件一览

- `scalar.yaml`：标量（Byte / Short / Int / Long / Float / Double / String 及布尔字面量）的类型严格性与等值语义；
- `compound.yaml`：复合标签的子集语义、嵌套、以及中文键与引号键；
- `list.yaml`：列表的存在量词语义、顺序无关、集合去重与空列表不对称性；
- `mixed.yaml`：多类型混合的综合用例。

### 注意

- 本目录用例不含 `[B;...]` / `[I;...]` / `[L;...]` 数组 SNBT。数组 SNBT 的端到端匹配由 `TestNBTMatcherSNBT.arraySNBTEndToEndTest` 覆盖，数组匹配语义由 `TestNBTMatcherArray` 直接用 Java API 构造覆盖。
- SNBT 字符串在 YAML 里一律用单引号包裹，文件编码 UTF-8、无 BOM。
- "空字符串键"无法经 SNBT 表达（`{"":1}` 在解析侧报 EMPTY_KEY）：空键是保留语义——异构列表以空键把非复合值包裹成复合标签作容器，故不向 SNBT 文本开放。本数据集因此只覆盖空字符串值；空键的匹配语义如需覆盖，须用 Java API 直接构造。引号键不剥离空白，纯空白键与带空格键都能经 SNBT 表达，见 `compound.yaml` 的带空格键用例。
