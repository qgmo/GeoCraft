# 测试：流体物理-通用-搜索流体

## 测试描述

该测试用于验证流体搜索算法（`FluidSearchUtil.findFluid`）的正确性。

流体搜索算法采用广度优先搜索（BFS），从指定起始位置出发，在可通行的方块中搜索流体方块。算法具备以下能力：

- 指定目标流体类型（不指定时匹配任意流体）；
- 限制仅在水平面内搜索（`flat` 模式）；
- 找到流体后优先向上寻找最高的连通流体方块（`up_if_possible` 模式）；
- 设置最大迭代次数限制搜索范围；
- 排除特定位置上的流体不计入搜索结果。

搜索过程中，BFS 仅向可通行（passable）、可替换（replaceable）或流体方块扩展，不会穿过固体方块。水平方向上不会往来路方向回头搜索。

## 样例格式

输入格式为一个描述结构的格文件。在格文件的元数据扩展（ext）处，需填写以下字段：

- `begin_at`：一个三元整数组，表示搜索的起始坐标。
- `max_iter`：正整数，BFS 的最大迭代次数。超过此次数的节点会被跳过。
- `expected`：布尔值，期望是否能找到流体（`true` = 能找到，`false` = 找不到）。

以下字段为可选：

- `target`：词块字符串，指定要搜索的流体类型。不填时搜索任意流体。
- `flat`：布尔值，默认 `false`。为 `true` 时仅在水平面搜索，不向上扩展。
- `up_if_possible`：布尔值，默认 `false`。为 `true` 时，找到流体后沿竖直方向向上寻找最高的连通流体方块。
- `ignored`：坐标列表，搜索时忽略的位置。这些位置上的流体不会被视为搜索结果。

## 样例 #1

### 输入 #1

```
---
ver: 1
default: 石
ext:
  begin_at: [2,2,2]
  max_iter: 11451
  expected: true
---
略

石石石石石石石
石〇〇〇〇〇石
石石石石石〇石
石石石石石〇石
石水〇〇〇〇石
石石石石石石石

略
```

## 样例 #2

### 输入 #2

```
---
ver: 1
default: 石
ext:
  begin_at: [2,2,2]
  target: 火
  max_iter: 100
  expected: false
---
略

石石石石
石〇水石
石石石石

略
```

## 说明/提示

### 样例 1 解释：

起始坐标 `[2,2,2]` 位于第二层、第二行、第二列的空气方块处。石墙围成的空间内有一条倒 U 形的空气通道：沿第二行向右到达最右列，再沿右侧向下穿过第三、四行，最终到达第五行第二列的水方块（`水`，Level 3 静态水）。搜索算法通过 BFS 沿此通道找到水，因此期望结果为 `true`。

### 样例 2 解释：

搜索目标为岩浆（`火`），但场景中只有水（`水`）。虽然 BFS 能抵达水方块，但流体类型不匹配，因此期望结果为 `false`。
