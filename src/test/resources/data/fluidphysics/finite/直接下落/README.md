# 有限水体测试数据格式说明（直接下落）

本目录中的测试文件用于验证有限原版流体在“直接下落”流动规则下的行为。

每个测试用例由两个文件组成：

```
<name>.in   初始场景
<name>.ans  期望结果
```

测试程序会读取 `.in` 文件构建初始世界，并执行且仅执行**一次**流体直接下落更新，然后将结果与 `.ans` 文件中的结构进行比较。
测试世界的外部默认全部填满黑曜石。计划刻更新不存在，但可能触发**邻居更新**。

---

# 输入文件格式

输入文件描述模拟开始前的世界状态。

第一行包含两个整数：

```
h n
```

含义：

* `h`：结构高度（层数）
* `n`：每层结构尺寸，为 `n × n`

第二行包含三个整数：

```
x y z
```

表示触发水体更新的位置坐标。

随后给出结构数据，共 `h` 层。

每一层由 `n` 行组成，每行 `n` 个中文字符，每个字符表示在特定位置的一个方块状态。
层与层之间使用一个空行分隔。
每一层的第 `i` 行表示 `z` 坐标为 `i-1` 的方块，每一行从左到右的第 `j` 个字符表示 `x` 坐标为 `j-1` 的方块。  

输入结构中的空白字符会被忽略。

# 输出文件格式

输出文件描述模拟结束后的世界结构。

格式与输入中的结构部分完全相同：

* 共 `h` 层
* 每层 `n × n`
* 层之间使用空行分隔

字符表示含义与输入一致。

输入和输出文件均可以在结构行之后，补充任何内容，例如注释，程序不会解析这一部分内容。

# 输入输出样例

## 输入

```
2 3
1 1 1
石石石
石灬石
石石石

```

## 输出

```
石石石
石溢石
石石石

石石石
石〇石

```

## 解释

输入定义了位于 (1,1,1) 的流体开始下落，(1,1,1) 的流体是灬，也就是四层水，四层水下落和下方的灬合成八层**流动水**，即**溢**。
